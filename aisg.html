Análise Interativa: Padrão AISG v3.0

Uma exploração das mudanças e novos recursos.

Esta seção fornece uma introdução de alto nível ao padrão AISG (Antenna Interface Standards Group) e as motivações por trás da versão 3.0. Explore os cartões abaixo para entender as principais mudanças que definem esta atualização significativa, focada em suportar as demandas de redes móveis modernas, como 5G e antenas complexas (MIMO).

O que é AISG?

AISG é um padrão global para a interface de controle entre uma estação base (BTS) e dispositivos de linha de antena (ALDs), como RETs (Remote Electrical Tilt) e TMAs (Tower Mounted Amplifiers). Ele permite que a estação base configure, monitore e gerencie esses dispositivos remotamente, geralmente sobre o mesmo cabo de RF ou um cabo dedicado.

Por que a v3.0?

A versão 2.0 (baseada em RS-485) era suficiente para antenas simples, mas lenta e limitada para sistemas modernos. O 5G, MIMO Massivo e antenas ativas exigem gerenciamento de mais dispositivos, configurações mais complexas e velocidades de comunicação muito maiores. A v3.0 foi criada para endereçar esses desafios, introduzindo principalmente o suporte a IP.

Principais Inovações da v3.0

Protocolo IP (eAISG)

A mudança mais significativa. A v3.0 introduz o "eAISG", permitindo que os comandos AISG sejam encapsulados em pacotes UDP/IP. Isso permite a comunicação via Ethernet padrão.

Aumento Drástico de Velocidade

Enquanto o RS-485 (v2.0) opera a 9.6 kbps (ou 38.4 kbps), a nova camada Ethernet (v3.0) suporta 10/100 Mbps. Isso é crucial para baixar software e configurar antenas complexas rapidamente.

"Ganging" (Multi-ALD)

Define uma forma padrão de um único dispositivo físico (como uma antena multibanda) expor múltiplas unidades lógicas de controle (ALDs). Simplifica o gerenciamento de sistemas de antenas complexos.

A transição da versão 2.0 para a 3.0 não foi apenas uma atualização, mas uma modernização fundamental. A introdução do suporte a IP, conhecida como "eAISG" (Extended AISG), é a pedra angular da v3.0. Esta seção detalha como a nova pilha de protocolos funciona. Em vez de se limitar a uma conexão serial RS-485, a v3.0 permite que os dispositivos de antena sejam cidadãos de primeira classe em uma rede IP, simplificando a integração, melhorando a velocidade e permitindo um gerenciamento muito mais robusto e flexível.

Nova Pilha de Protocolos (eAISG)

A v3.0 define como os comandos AISG existentes são encapsulados para transporte sobre redes IP. A camada de aplicação AISG (com seus comandos como `SetTilt`, `Scan`, etc.) permanece a mesma, garantindo a compatibilidade. O que muda é o transporte, como ilustrado abaixo.

Ethernet (10/100Base-T)

Embora o conector físico AISG (o circular de 8 pinos, conforme IEC 60130-9, similar ao da imagem que você forneceu) permaneça o mesmo, a *utilização* desses pinos evoluiu na v3.0 para suportar Ethernet. Esta seção permite que você alterne entre as duas principais configurações de pinagem para entender como o mesmo conector pode suportar tanto o legado RS-485 quanto a nova comunicação baseada em IP.

Pinagem do Conector AISG (Macho/Fêmea 8-pin)

Configuração Padrão (RS-485)

Esta é a pinagem clássica do AISG v2.0, usando um par para dados RS-485 e pinos dedicados para alimentação DC.

  • Pino 1: +10-30V DC (Alimentação)
  • Pino 2: Não usado
  • Pino 3: RS-485 B
  • Pino 4: Não usado
  • Pino 5: RS-485 A
  • Pino 6: Não usado
  • Pino 7: DC Retorno (GND)
  • Pino 8: Não usado

Configuração v3.0 (Ethernet + RS-485)

Para suportar Ethernet e manter a compatibilidade, a v3.0 utiliza os pares não usados anteriormente. Esta é uma implementação "híbrida" comum que permite que o mesmo cabo transporte DC, RS-485 e Ethernet 10/100Base-T simultaneamente.

  • Pino 1: +10-30V DC (Alimentação)
  • Pino 2: Ethernet RX-
  • Pino 3: RS-485 B
  • Pino 4: Ethernet TX+
  • Pino 5: RS-485 A
  • Pino 6: Ethernet TX-
  • Pino 7: DC Retorno (GND)
  • Pino 8: Ethernet RX+

Nota: Esta é uma pinagem comum; outras configurações, como PoE (Power over Ethernet), também são possíveis dentro da especificação v3.0, que podem usar os pinos de dados para alimentação.
